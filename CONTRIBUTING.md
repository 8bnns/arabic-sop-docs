# دليل المساهمة

شكراً لاهتمامك بالمساهمة في هذا المشروع! هذا الدليل يساعدك على فهم كيفية إضافة SOPs جديدة أو تحديث الموجودة.

---

## قبل البدء

1. اقرأ `README.md` لفهم نطاق المشروع
2. راجع الوثائق الموجودة لفهم الهيكل والأسلوب
3. تأكد من أن لديك حساب GitHub وتم ربط git على جهازك

---

## إضافة SOP جديد

### 1. أنشئ فرعاً جديداً
```bash
git checkout -b sop/اسم-الإجراء
```

### 2. انسخ قالب SOP
استخدم أحد الملفات الموجودة كقالب واتبع هذا الهيكل:

```markdown
# إجراء التشغيل الموحد (SOP) — اسم الإجراء

**الرقم المرجعي:** SOP-XXX  
**التاريخ:** [الشهر السنة]  
**المسؤول:** [الفريق/الشخص]  

---

## الغرض
[وضح الهدف بشكل واضح ومختصر]

## النطاق
[من يستخدم هذا الإجراء؟ على أي أنظمة؟]

## المتطلبات الأولية
- عنصر 1
- عنصر 2

---

## خطوات التنفيذ

### 1. الخطوة الأولى
- تفصيل
- تفصيل

### 2. الخطوة الثانية
```bash
أوامر shell إن وجدت
```

---

## معايير الإتمام
- [ ] معيار 1
- [ ] معيار 2

---

## التكرار
[يومي/أسبوعي/شهري]

---

*آخر تحديث: [التاريخ]*
```

### 3. حدّث الفهرس
أضف الإجراء الجديد إلى `فهرس_إجراءات_التشغيل_الموحدة.md`:

```markdown
### SOP-XXX — اسم الإجراء
**الملف:** [SOP_اسم_الإجراء.md](./SOP_اسم_الإجراء.md)  
**التكرار:** [التكرار]  
**الوصف:** [وصف مختصر]
```

---

## تحديث SOP موجود

### 1. أنشئ فرعاً جديداً
```bash
git checkout -b update/SOP-XXX-اسم-التحديث
```

### 2. عدّل الملف المناسب
- احتفظ بالهيكل الأساسي
- حدّث سجل التحديثات في الفهرس
- غيّر تاريخ "آخر تحديث"

### 3. اشرح التغييرات في رسالة الـ commit
```bash
git commit -m "تحديث SOP-XXX: وصف التغيير

- تفصيل التغيير 1
- تفصيل التغيير 2

Co-Authored-By: Oz <oz-agent@warp.dev>"
```

---

## معايير الجودة

جميع الوثائق يجب أن تحقق:

- ✅ **الوضوح:** اللغة بسيطة وواضحة
- ✅ **الاكتمال:** جميع الأقسام مملوءة
- ✅ **الدقة:** المعلومات صحيحة ومختبرة
- ✅ **التنسيق:** متسق مع الوثائق الأخرى
- ✅ **الأوامر:** تُختبر قبل النشر

---

## عملية المراجعة

1. ادفع فرعك وافتح Pull Request
2. أضف وصف واضح للتغييرات
3. انتظر مراجعة الكود
4. رد على التعليقات وأجرِ التعديلات المطلوبة
5. بعد الموافقة، دمّج الفرع

---

## أسئلة شائعة

**س: كيف أختار رقم SOP؟**  
ج: استخدم الرقم التسلسلي التالي (مثلاً إذا كان آخر SOP هو 004، استخدم 005).

**س: ما أفضل طول لـ SOP؟**  
ج: 30-80 سطر تقريباً. يجب أن تكون مختصرة لكن شاملة.

**س: هل يمكنني تحديث SOP الموجودة؟**  
ج: نعم! افتح Issue أو PR بشرح التغيير المطلوب.
