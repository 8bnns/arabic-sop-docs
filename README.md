## 🗂️ هيكل كل وثيقة

كل وثيقة SOP تتضمن الأقسام التالية:

- **الغرض** — هدف الإجراء
- **النطاق** — من يُطبّق هذا الإجراء وعلى ماذا
- **المتطلبات الأولية** — ما يجب توفّره قبل البدء
- **خطوات التنفيذ** — تفصيلية مرقّمة مع أوامر shell عند الحاجة
- **معايير الإتمام** — قائمة تحقق للتأكد من اكتمال العملية
- **التكرار** — جدول تنفيذ الإجراء

---

## 🚀 كيفية الاستخدام

1. افتح الفهرس الشامل لاختيار الإجراء المناسب
2. اقرأ المتطلبات الأولية وتأكد من استيفائها
3. اتّبع خطوات التنفيذ بالترتيب
4. ضع علامة ✅ على كل بند في قائمة معايير الإتمام
5. وثّق نتيجة العملية في `سجل_العمليات`

## 🤝 المساهمة

لإضافة إجراء جديد أو تحديث إجراء موجود:
1. أنشئ فرعاً جديداً: `git checkout -b sop/اسم-الإجراء`
2. أضف أو عدّل الملف المناسب
2. حدّث `فهرس_إجراءات_التشغيل_الموحدة.md`
3. افتح Pull Request مع وصف واضح للتغييرات
